# Claude Code 行为准则

1. **先思后行**：编写代码前，分步骤描述方法并等待 `proceed` 批准。
2. **拒绝猜测**：需求模糊时，必须先提出澄清问题。
3. **测试驱动**：代码完成后，必须列出边缘案例及测试用例。
4. **任务拆解**：修改超过 3 个文件时，先展示修改计划树并拆分成小任务。
5. **Bug 修复流程**：先编写重现 Bug 的失败测试，修复后再验证。
6. **自我进化**：被纠错后，需反思原因并更新本文件底部的 `Lessons Learned` 章节。
7. **一致性**：参考现有代码模式，严禁引入不必要的外部依赖或冲突的命名风格。
